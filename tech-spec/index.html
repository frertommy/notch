01 · System Overview
Five systems. One currency.

Notch v1 is built on five interlocking systems, all denominated in NP (Notch Points). NP is earned through trading, quest completion, league wins, streaks, and referrals — and accumulates as the lifetime measure that determines rank.

Every action funnels NP into a single lifetime counter. Lifetime NP determines rank. Rank is a permanent visible signal of player history. Spending mechanics (sinks, fee discounts, store) are deferred to v3 — see Section 11.
02 · NP — Currency Model
Two counters. Same currency.

Every user has two NP numbers tracked at all times. They behave differently and are never confused.

Lifetime NP
Monotonically increasing. Only goes up.
Determines current rank. Never decreases — even when v3 sinks ship and users spend Balance NP, lifetime stays intact. This is the airdrop allocation meter and the status meter at once.
NP Balance
Spendable counter.
In v1, balance and lifetime move together because there are no sinks yet. Starting v3, sinks (Store, fee discounts) decrement balance only. Architecture for split is built day-one even though the spend hasn't shipped.

09 · Anti-farming
Soft currencies die to bots. Guardrails everywhere.
Daily NP ceiling
30,000 NP / user / day across all sources combined. Catches outliers automatically.
Per-source caps
Trading 10K/day · Streak 100/day × 30 max · Sport quests 5K/wk per sport
Wash trade detection
Open + opposite-side same team within 5 min at <$200 notional → volume excluded from NP calc
Multi-account fraud
Same KYC / device fingerprint / payment method linked → all linked NP zeroed · repeat offense = ban
Leverage cap on NP-volume
Notional credited at min(actual_leverage, 5×) for NP calc only. Trading still uses real leverage.
Minimum hold time
Position must be held ≥30 min to count toward NP. Kills open-close-open scalping spam.
Multi-day rule
Volume must accrue across ≥3 distinct calendar days for milestone unlocks beyond first $1K. Stops single-session farm runs.
Referral KYC gate
Referee KYC + first $100 notional trade required before any payout releases to referrer.
10 · Data flow · what triggers what
Everything is downstream of one event: trade close.

Position open is an intent. Position close (or settlement) is the value-realizing event. Every NP, quest progress, league standing, referral payout, and rank-up is recomputed off this single trigger.

All NP grants flow through the same validator. There is one place to fix bugs in the volume math, one place to handle anti-farm rules, one place to grant NP. Everything downstream consumes the validated grant event.
11 · Deferred · v2 / v3 backlog
What's NOT shipping in v1.

Architectural placeholders are present from day one (lifetime/balance split, rank-multiplier hooks, currency abstractions) so adding these later doesn't require a migration.

v2 · Social
League chat · DMs · global feed · public profiles (notch.win/u/username) · trade share cards · rank multiplier reactivation · indirect referrals · rank-decay on referral tail.
v3 · Economy
Credits as second currency (fee offsets) · Store with redemptions (merch · WC raffle · Athletic subscription · cosmetic borders) · Stake-to-use fee discounts · Weekly trading rewards pool funded from fees.
Token optionality
Lifetime NP earned (not balance) determines $NTCH allocation if and when the token launches. Snapshot mechanic preserved — no commitment to launch, no commitment to ratio. Polymarket-style ambiguity by design.
